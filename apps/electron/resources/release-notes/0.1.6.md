# v0.1.6 - Agent-authored Excalidraw canvases

## Features

- **Agent-authored Excalidraw canvases** - Agents can now generate editable, file-backed canvas diagrams - architecture maps, flow charts, system diagrams - directly in the chat. You describe what you want; the agent emits a coordinate-free graph (nodes, edges, direction) and the app lays it out automatically and applies the Graphite design language - neutral ink-line styling with accent color only where it carries meaning - then renders it as a real Excalidraw scene you can open fullscreen, edit, and rearrange. Canvases live in the session and re-render in place as the agent refines them.

## Improvements

- **Pi SDK uplift (upstream v0.10.4 merge)** - The Pi backend moved to `@earendil-works/pi-ai`, which pulls new vendor models from its models.dev catalog automatically as they land upstream - so newly added providers/models show up without a fork change.
- **In-app "What's New" shows this fork's releases** - The What's New changelog now lists this fork's own versioned release notes instead of upstream's.
- **Automation approval workflow guidance** - The bundled automations guide, edit popover prompt, and session config validation tool description now teach agents the intended `prompt` -> `confirm` -> follow-up action pattern, covering both webhook/API follow-ups and same-session prompt follow-ups.
- **Automation approval gate behavior** - Approval workflows now reject ambiguous `confirm`-first or multi-action pre-confirm shapes, keep follow-up actions attached across chained confirmation gates, and show Run Test as waiting for confirmation instead of passed when a card is still pending.

## Bug Fixes

- **Text-only models are no longer told to "view" a canvas preview they can't see** - The Excalidraw guidance unconditionally told the model to open the rendered preview PNG to review the diagram. For text-only models (no image input) that was a dead end that also wasted a tool round-trip; they now get a structural self-check (via `excalidraw_describe_canvas`) instead, while image-capable models keep the visual review.
- **Canvas zoom controls no longer overlap the fullscreen/edit chrome** - Moved the canvas zoom controls clear of the fullscreen and edit toggles so they no longer collide.

## Breaking Changes

- None.
